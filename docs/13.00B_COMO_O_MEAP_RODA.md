# 13.00B — Como o MEAP roda (do dado à zona oficial)

> Descritivo do pipeline real de geração de Zonas de Manejo (MEAP M2).
> Complementa o 13.00 (conceito) e o 13.00A (revisão do fluxo).

---

## Princípio: o MEAP NÃO interpola

O MEAP **não** gera mapa de atributo. Ele **agrupa por similaridade** mapas que
JÁ foram interpolados em outros módulos (fertilidade, NDVI…). Cada pixel é um
"perfil" de vários atributos; pixels parecidos viram a mesma zona.

```
Mapas já interpolados (fertilidade, NDVI…)
        │  (co-registro na mesma malha)
        ▼
Empilhar camadas  →  cada pixel = vetor [cam1, cam2, …]
        ▼
Normalizar (z-score por camada) × pesos
        ▼
Clusterizar (Fuzzy c-means / K-means)
        ▼
FPI / NCE para 2..12 zonas  →  sugestão do nº
        ▼
Gerar com o nº ESCOLHIDO  →  classe por pixel
        ▼
Vetorizar + área mínima + ordenar Alta→Baixa
        ▼
ZONAS OFICIAIS (classes) · POLÍGONOS (manchas)
```

## 1. Camadas (entrada)

`lib/meap/gerar.ts carregarCamadas`:
- Lê os **grids salvos** da fertilidade (cada nutriente × profundidade) e os **NDVI
  mantidos** (Sentinel/CBERS).
- **Co-registra** tudo na MESMA malha de referência (a da fertilidade; se não houver,
  a do NDVI mais fino, teto de 160 px/lado). NDVI é reamostrado (bilinear NaN-aware).
- Resultado: N camadas alinhadas pixel a pixel (mesmo bounds/shape).

## 2. Configurar (Etapa 1)

Você escolhe **quais camadas** entram, o **peso** de cada uma (0 = ignora · 1 = padrão ·
↑ = manda mais) e o **método** (Fuzzy c-means ou K-means).

## 3. Analisar (Etapa 2) — `/zonear-analisar` → `interp.analisar_multi`

1. **Empilha**: monta a matriz `X` (linhas = pixels com dado em TODAS as camadas;
   colunas = camadas).
2. **Normaliza**: z-score por coluna (`_normalizar_colunas`) — assim argila, CTC e NDVI
   entram em escala comparável. Depois multiplica pelo **peso** de cada camada.
3. **Clusteriza** com Fuzzy c-means (`_fcm`) para **cada nº de 2 a 12**.
4. **FPI / NCE** (Fridgen et al. 2004 / método MZA), por nº de zonas:
   - `FPI = 1 − (c·F − 1)/(c − 1)`, com `F = (1/n)·ΣΣ u²` (coef. de partição).
   - `NCE = H / ln(c)`, com `H = −(1/n)·ΣΣ u·ln(u)` (entropia de partição).
   - Ambos **menores = melhor** (zonas mais nítidas, menos "difusas").
5. **Sugestão**: o nº com a melhor combinação FPI+NCE; a distância para o 2º melhor
   vira a **confiança** (%). Você vê o gráfico FPI×NCE (estilo cotovelo) e escolhe o nº
   (a sugestão não é obrigatória).

## 4. Decidir e gerar (Etapas 3–4) — `/zonear-gerar` → `interp.gerar_multi`

1. **Clusteriza** com o nº ESCOLHIDO (FCM → classe de cada pixel pela maior pertinência;
   ou K-means).
2. **Área mínima** (`_sieve_labels`, estilo `gdal_sieve`): funde manchas conexas menores
   que o limite na zona vizinha majoritária (itera até estabilizar).
3. **Vetoriza**: cada mancha contígua vira um **polígono**.
4. **Ordena Alta→Baixa**: usa as camadas de "potencial" presentes
   (`RANK_SIMBOLOS` = produtividade/NDVI/MO/CTC) para decidir qual classe é a de maior
   potencial; se não houver, usa o conjunto das camadas.

## 5. Avaliar (Etapa 5) — front (`MeapSection`)

- **Limpa resquícios**: remove buracos/ilhas menores que a área mínima (`lib/meap/fundir.ts`).
- **Agrupa por classe = ZONAS OFICIAIS**: o nº de zonas oficiais = o nº escolhido. Cada
  mancha é um **POLÍGONO** (uma zona pode ter vários). Ex.: "6 zonas oficiais · 9 polígonos".
- **Cores** em gradiente verde→vermelho por potencial; **rótulos** legíveis
  (Muito alto…Muito baixo); por zona: nº de polígonos + área + menor/maior.
- **CV (homogeneidade)** por zona: pega os pontos de **laboratório** que caem dentro de
  cada zona e calcula o CV da variável de validação (argila/CTC…). Zona boa = CV baixo.
- **Edição**: fundir polígonos manualmente; **camada de fundo** + slider de opacidade
  para comparar a zona com NDVI/fertilidade/EC por baixo.
- **Salvar zoneamento** → marcar **Padrão** → vai para a **Amostragem** (grid por zona) e
  vira o **Ambiente Adotado** (bloco M1 no topo).

## Resumo de quem faz o quê

| Onde | Arquivo | Papel |
|---|---|---|
| Carregar/co-registrar camadas | `lib/meap/gerar.ts` | front |
| Analisar (FPI/NCE) | `backend/interp.py analisar_multi` | motor |
| Gerar (cluster+sieve+vetor) | `backend/interp.py gerar_multi` | motor |
| Avaliar/editar/salvar | `components/talhao/MeapSection.tsx` | front |
| CV por zona | `lib/meap/cv.ts` | front |
| Adoção (Ambiente M1) | `lib/meap/adocao.ts` | front |

## O que o MEAP NÃO faz (e onde isso é feito)

- **Interpolar** um atributo (krigagem/IDW): é dos módulos Fertilidade / Condutividade /
  NDVI. O MEAP só **consome** esses mapas. Por isso, para a Condutividade entrar como
  camada do MEAP, primeiro o mapa de EC precisa estar interpolado e salvo.
