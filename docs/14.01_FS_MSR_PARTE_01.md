# 14.01 — FS MSR · Parte 01 — NDVI Sentinel-2 (Fase S1)

> Especificação funcional da **V1 do módulo NDVI / Satélite**. Âncora: `14.00_DOC_CONCEITUAL_MSR.md`.
> Status: **em implementação** (aprovada em 2026-06-27).

---

## 1. Objetivo da fatia

Dado um talhão, buscar a **cena Sentinel-2 mais recente com pouca nuvem**, calcular o **NDVI** no backend, exibi-lo no mapa com a legenda oficial e **salvar** o raster datado. Sem upload, sem multi-índice, sem nuvem de produção (tudo no backend local, como Fertilidade/MEAP).

---

## 2. Backend

### 2.1 Módulo `backend/msr.py`
Motor de Sensoriamento Remoto isolado (não toca em `interp.py`).

- `gerar_ndvi(polygon_geojson, data_ini, data_fim, nuvem_max=40.0, pixel_m=10.0) -> dict`
  1. `poly = shape(geojson)`, `bbox = poly.bounds`.
  2. **STAC** (`pystac-client`): catálogo `https://earth-search.aws.element84.com/v1`, coleção `sentinel-2-l2a`, filtro `bbox + datetime(data_ini/data_fim) + eo:cloud_cover < nuvem_max`. Pega a cena **mais recente** (ordena por `datetime` desc).
  3. Bandas **red (B04)** e **nir (B08)** — lê só a janela do talhão e **reprojeta** para uma grade EPSG:4326 sobre os `bounds` (rasterio `reproject`, bilinear). Aplica `scale/offset` (`raster:bands`) → refletância.
  4. `NDVI = (nir − red)/(nir + red)`, recorta no polígono (NaN fora), clampa em [-1, 1].
  5. Devolve o **mesmo envelope da Fertilidade** + dados da cena:
     ```json
     {
       "bounds": [minx, miny, maxx, maxy],
       "grid": { "b64": "<Float32 norte-no-topo>", "shape": [rows, cols] },
       "stats": { "n", "min", "max", "media", "nx", "ny", "pixel_m", "indice": "NDVI" },
       "cena":  { "id", "data": "YYYY-MM-DD", "nuvem": <%>, "plataforma": "sentinel-2a" }
     }
     ```
- Deps importadas com guarda (`_HAS_MSR`): se faltarem (`rasterio`/`pystac-client`), o endpoint responde erro claro sem derrubar o backend.

### 2.2 Endpoint `backend/app.py`
- `POST /ndvi-sentinel` com `ReqNdvi { poligono, data_ini, data_fim, nuvem_max=40, pixel_m=10 }` → `interp`-style try/except (422 ValueError / 500 outros).
- `GET /health` passa a reportar `msr: _HAS_MSR`.

### 2.3 Dependências
`requirements.txt` += `pystac-client`, `rasterio`. (venv existente precisa de `pip install` manual — o `start.ps1` só instala na 1ª criação.)

## 3. Front

### 3.1 `src/lib/msr.ts`
Cliente puro (sem reescrever grid): `buscarNdviSentinel({ poligono, dataIni, dataFim, nuvemMax?, pixelM? }) -> RespNdvi`. Tipos `RespNdvi`/`CenaNdvi`. Reusa `Grid` de `fertilidade.ts`. Mesmo tratamento de "backend desligado".

### 3.2 `src/components/talhao/NdviSection.tsx`
- Polígono do talhão (igual Fertilidade: `uploadedGeo` → `talhao.geojson`).
- Controles: **período** (`dataIni` = hoje−60d, `dataFim` = hoje) + **nuvem máx** (default 40%) + botão **Buscar imagem (Sentinel-2)**.
- Render no mapa: reusa o overlay genérico (`setFertilidadeOverlay`) com `colorirGridComLegenda(grid, legNDVI)` (legenda `getLegendasPorAtributo('ndvi')[0]`).
- Mostra **info da cena** (data, satélite, nuvem%) + **estatísticas do raster** (média/mín/máx, grade, pixel) + barra da legenda NDVI.
- **Persistência** reusando `cloudSalvarMapa`/`cloudCarregarMapasPorPrefixo` com prefixo `${talhaoId}__ndvi__NDVI__${dataCena}` (grid gzip, guarda de 1 MB) — *tech debt S2: mover para coleção própria*.
- **Autoload** ao abrir: lista as datas salvas (série inicial), mostra a mais recente; clicar numa data troca o mapa.

### 3.3 Aba
`TalhaoPage.tsx`: aba `ndvi` passa a `pronto: true` e renderiza `<NdviSection />` (entre Compactação e Arquivos, onde já está).
