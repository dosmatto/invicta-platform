# 13.02 — FS MEAP · Parte 02 — Motor Matemático (Camadas → Modelo → Zonas)

> Documento vivo. Versão inicial: 2026-06-25.
> Pré-requisitos: `13.00` (conceito) e `13.01` (dados). Termos: `13.99`.
> Esta parte define **como** o MEAP transforma camadas oficiais em zonas e calcula convergência
> e confiabilidade. Backend de cálculo: `backend/` (FastAPI + PyKrige) — ver `lib/fertilidade.ts`.

---

## 1. Escopo

Pipeline matemático completo, da ingestão das camadas oficiais até a geração das zonas e o
cálculo das métricas que aparecem na tela (convergência, mudança entre versões, confiabilidade,
índice de convergência por zona). Ajustes operacionais (área mínima, suavização para campo) são
da **13.03**.

---

## 2. Pipeline (visão geral)

```
[1] Ingestão        →  camadas oficiais elegíveis (13.01 §4.5)
[2] Gate de qualidade → descarta camadas insuficientes; define peso
[3] Camadas JÁ interpoladas → consome os grids prontos (o MEAP NÃO re-interpola)
[4] Co-registro + normalização → mesma malha + z-score por camada
[5] Empilhamento    →  cada pixel vira um vetor de atributos das camadas
[6] Clusterização   →  k-means / fuzzy c-means (similaridade); nº de zonas por FPI/NCE
[7] Vetorização     →  raster de rótulos → polígonos (modelo MATEMÁTICO)
[8] Matching de DNA →  casa zonas novas com linhagens existentes (13.01 §5)
[9] Métricas        →  convergência, mudança, confiabilidade, CV por zona, redução de variância
```

A saída de [7]+[8]+[9] é uma **VersaoModelo** com suas **ZonasDeManejo** (geometria matemática).
A 13.03 recebe isso e produz a geometria operacional e a Zona Oficial.

---

## 3. Camadas oficiais e pesos

### 3.1 Gate de qualidade (passo [2])

Uma camada só entra no modelo se `elegivel = true`. Critérios mínimos por tipo:

| Camada | Critério de elegibilidade (mínimo) | Qualidade ↑ quando |
|---|---|---|
| Fertilidade | ≥ 1 importação de lab com cobertura espacial do talhão | mais pontos, recente |
| Amostragem | grade com densidade ≥ limiar (ha/ponto) | densidade maior |
| Produtividade | ≥ 1 mapa de colheita limpo | ≥ 3 safras (habilita "média 3 anos") |
| NDVI | ≥ 1 cena sem nuvem no período | série temporal |
| Compactação | malha mínima de pontos | malha densa |
| Topografia | DEM disponível | resolução maior |

### 3.2 Pesos

Cada camada recebe `peso ∈ [0,1]` (default por empresa, ajustável por talhão). Os pesos das
camadas elegíveis são **renormalizados para somar 1** antes da fusão. Pesos default sugeridos
(parametrizáveis):

```
fertilidade 0.30 · produtividade 0.30 · topografia 0.15 · ndvi 0.15 · compactacao 0.10
```

> Produtividade e fertilidade dominam por serem os indicadores mais diretos de *potencial*.

---

## 4. Normalização (passo [3])

- Reprojeção de todas as camadas para uma **grade-alvo comum** (resolução parametrizável,
  default compatível com a Fertilidade).
- Cada variável contínua é escalada para `0..1` por *min-max robusto* (percentis 2–98, para
  resistir a outliers).
- Variáveis "quanto-maior-pior" (ex.: compactação) são **invertidas** para que `1 = melhor`.

---

## 5. Camadas já interpoladas (passo [3]) — o MEAP NÃO re-interpola

**Princípio (decisão do usuário):** a zona de manejo é feita por **similaridade/clusterização
dos mapas que JÁ foram interpolados** pelos respectivos módulos — não é o MEAP que interpola.

- A **interpolação** (krigagem/IDW) é responsabilidade de cada módulo: a Fertilidade já gera e
  salva os grids (`carregarGridsTalhao`); Produtividade e NDVI farão o mesmo. O MEAP **consome
  os grids prontos** e os usa como **camadas**.
- Re-interpolar só como **fallback** quando uma camada selecionada ainda não tiver grid.
- A krigagem em si (semivariograma esférico/exponencial/gaussiano + validação cruzada LOO) vive
  no `backend/` (`interp.gerar_grid`) e é reaproveitada pelos módulos. Referência de estudo:
  [SmartMapPlugin](../references/smartmap/SCRIPTS_APROVEITAVEIS.md) (reescrito, nunca importado).

---

## 6. Fusão multicamada (passo [5])

Índice de potencial contínuo `P(x)` por célula:

```
P(x) = Σ_c  peso_c · valorNormalizado_c(x)        (Σ peso_c = 1)
```

Células sem dado em alguma camada usam a média da camada (imputação neutra) e **reduzem a
confiabilidade local** (entra no cálculo de confiabilidade, §9). `P(x) ∈ [0,1]`.

---

## 7. Clusterização → zonas (passo [6])

Análise de **similaridade**: empilhadas e normalizadas (z-score) as camadas escolhidas, cada
pixel é um **vetor de atributos**; pixels parecidos caem na mesma zona.

- **Dois algoritmos, ambos disponíveis (decisão do usuário):** **k-means** (rígido) e
  **fuzzy c-means / FCM** (difuso, com grau de pertinência por pixel). FCM é também a base dos
  índices FPI/NCE. Implementação: k-means via `scipy`, FCM reimplementado em numpy
  (`backend/interp.zonar_multi`).
- **Nº de zonas pelos índices FPI/NCE (Fridgen et al. 2004 / MZA):** roda-se o FCM para uma
  faixa de nº de classes (ex.: 2..6) e, para cada um:
  - **FPI** (Fuzziness Performance Index) `= 1 − (c·F − 1)/(c − 1)`, `F` = coef. de partição `(1/n)ΣΣu²`;
  - **NCE** (Normalized Classification Entropy) `= H / ln(c)`, `H` = entropia `−(1/n)ΣΣ u·ln(u)`.
  - O **nº ótimo é onde FPI e NCE são MÍNIMOS** (menor sobreposição / maior organização). A
    plataforma **sugere o mínimo**; o usuário decide pela **curva (gráfico FPI/NCE)**.
- As classes são **ordinais** (Zona 01 = maior potencial). A ordenação fina (manual + sugerida)
  é da 13.03 §2.1.
- Referência de estudo: FCM/FPI/NCE do [SmartMapPlugin](../references/smartmap/SCRIPTS_APROVEITAVEIS.md)
  (que usa `skfuzzy`), **reimplementados** na Invicta.

## 8. Vetorização e matching de DNA (passos [7]–[8])

- Raster de classes → polígonos (conversão raster→vetor), gerando a **geometria matemática**.
- **Matching de DNA** (algoritmo do 13.01 §5):

```
similaridade(zonaNova, dna) =
      α · (1 − distânciaAssinatura)        // vetor agronômico normalizado
    + β · (1 − distânciaCentroideNormalizada)
    + γ · sobreposiçãoEspacial(zonaNova, últimaGeometriaDoDNA)

casar se  similaridade ≥ LIMIAR_DNA   (default 0.6; α=0.5, β=0.2, γ=0.3)
senão     criar novo DNA (zona nova)
DNA sem zona casada na versão → marcar 'dormente'
```

Determinismo: seed fixa por versão (13.01) ⇒ resultado reproduzível.

## 9. Métricas (passo [9])

### 9.1 Convergência do Modelo (a "76%")

Mede estabilidade espacial entre a versão atual e a anterior:

```
mudancaEntreVersoes = (área que trocou de classe) / (área total)      → "2,8%"
convergencia        = 1 − mudancaEntreVersoes_normalizada             → "76%"
```

> `convergencia` é suavizada por uma média das últimas K transições (default K=3) para não
> oscilar com um único reprocesso. Primeira versão: `convergencia = 0` (sem anterior).

### 9.2 Índice de Convergência por Zona (a "82%")

Por linhagem de DNA: estabilidade da geometria/atributos da zona entre versões (Jaccard de
área + estabilidade da assinatura).

### 9.3 Confiabilidade

Função de **densidade e qualidade dos dados** (não da convergência):

```
confiabilidade = f( densidadeAmostral, nº de camadas elegíveis,
                    recência dos dados, cobertura espacial sem imputação )
→ faixa: baixa | média | alta
```

### 9.4 Estado do Conhecimento

Agregado qualitativo por talhão: `insuficiente → inicial → bom → excelente`, função do número
de safras com dado, camadas elegíveis e histórico de versões.

### 9.5 CV por zona (homogeneidade interna) — **calculado na geração da versão**

> Esta é a métrica que valida se a zona "presta": **uma boa zona de manejo é internamente
> homogênea**. Calculada para **cada zona**, no momento em que a versão é gerada.

Para cada atributo `c` de **escala original** (com zero significativo — ex.: argila %, P, K,
produtividade sc/ha), dentro da geometria da zona:

```
CV_zona,c = ( desvio-padrão_c(zona) / média_c(zona) ) × 100        [%]
```

- **Atenção de escala (decisão de projeto):** o CV é calculado sobre os **atributos na escala
  medida**, **nunca** sobre o índice fundido `P(x)` normalizado 0..1 — min-max não tem zero
  significativo, então o CV do índice seria estatisticamente sem sentido.
- **Variável de validação (headline da zona):** por padrão **produtividade** (se houver ≥ 3
  safras); senão, o atributo elegível de **maior peso**. É o `cvValidacao` exibido na tela.
- **Demais atributos:** `cvPorAtributo` (ex.: argila 8%, P 24%) → vai para o prontuário e a
  explicabilidade ("homogênea em textura, variável em fósforo").
- **Faixa legível** (`homogeneidade`, limiares parametrizáveis):

| CV da variável de validação | Homogeneidade |
|---|---|
| ≤ 10% | Alta |
| 10–20% | Média |
| > 20% | Baixa (rever nº de classes / pesos) |

- **Robustez:** média/desvio por estatística resistente quando há outliers (winsorização nos
  percentis 2–98, coerente com a normalização do §4); ignorar células imputadas (§6) no cálculo.

### 9.6 Redução de variância (qualidade do zoneamento)

Métrica **do modelo** (não da zona): o quanto o zoneamento reduziu a variabilidade frente a
tratar o talhão como bloco único.

```
reducaoVariancia = 1 − ( variância média intra-zona / variância total do talhão )    [0..1]
cvMedioIntraZona = média ponderada por área do CV das zonas (variável de validação)
```

- `reducaoVariancia` alta ⇒ as zonas explicam bem a variabilidade do talhão (zoneamento bom).
- Alimenta o gate de **confiabilidade** (§9.3) e **guia a escolha de N** (§7, regra do cotovelo).

## 10. Parametrização (default + override)

```ts
interface ParametrosModelo {
  numClasses: number;            // default 5
  metodoZoneamento: 'quantis' | 'kmeans' | 'fuzzy';   // default 'quantis'
  pesos: Record<Evidencia['tipo'], number>;           // renormalizados p/ Σ=1
  resolucaoGrade: number;        // m/px da grade-alvo
  interpolacao: 'krigagem' | 'idw' | 'auto';          // default 'auto'
  limiarDNA: number;             // default 0.6
  seed: number;                  // reprodutibilidade
}
```

Defaults por **empresa** (Biblioteca/Configurações); override por talhão. Toda versão grava os
parâmetros usados (13.01 §4.3) — reprodutibilidade e auditoria.

---

## 11. Critérios de parada / quando recalcular

- **Gatilhos de novo cálculo:** nova camada oficial elegível, nova safra de produtividade,
  mudança de parâmetros, ou reprocesso manual.
- **Não recalcular** se nenhuma evidência nova entrou (evita versões idênticas inúteis).
- Consolidação (parar de evoluir significativamente) é decisão de **maturidade** → 13.03.

---

## 12. Saída desta parte

Uma `VersaoModelo` completa com zonas (geometria matemática), DNA casado e métricas calculadas,
pronta para o **Motor Operacional** (13.03) transformar em Zona Oficial.
