# 15.00 — Módulo de Condutividade Elétrica (CEa)

> **INVICTA AP — Variável Fixa do Talhão**
> Âncora da spec: `MD_Modulo_Completo_Condutividade_Invicta.md`.
> Bloco de docs 15.xx = Condutividade (como 13.xx = MEAP, 14.xx = MSR).

---

## Conceito

A condutividade elétrica aparente do solo (CEa) é uma característica **estrutural** do
talhão (correlaciona com argila, umidade e CTC). Depois de validada, fica **vinculada
permanentemente ao talhão** como **Variável Fixa**, podendo ter **várias versões** ao
longo do tempo — uma é a **oficial**. É uma das bases para as Zonas de Manejo (MEAP).

Integra-se a: Zonas de Manejo · Produtividade · Fertilidade · Textura · Altimetria ·
NDVI · Relatórios · Layouts.

## Princípio de reúso (DOC-001)

Reaproveita o que já existe — **não duplica**:
- **Import de pontos** = `lib/compactacao.ts` (`parseArquivoPontos`: SHP .zip / KML / GeoJSON / CSV / XLSX + mapeamento de colunas).
- **Interpolação** = motor da Fertilidade (`interpolar` → `backend/interp.py`), que já faz **krigagem automática** (testa esférico/exponencial/gaussiano, escolhe pelo RMSE da validação cruzada) + recorte no polígono + grid persistível.
- **Persistência** = nuvem `inv_mapas_fert` com namespace `condutividade__<talhao>__<versao>__<prof>` (grid gzip + autoload), igual à Fertilidade/Compactação.
- **Legenda oficial** = `sys_condutividade` (Biblioteca, escopo Sistema, editável).
- **Vetorização** = MEAP (a EC oficial vira fonte, como o NDVI).

## Fases

### Fase C1 — Fundação (FEITA)
Importar pontos (2 profundidades: rasa/profunda) → interpolar por profundidade
(krigagem automática) → **Índice de Qualidade** (Excelente/Boa/Regular/Baixa + apto
p/ MEAP, derivado do RMSE relativo da validação cruzada) → salvar como **Variável Fixa
do Talhão** (`inv_condutividade`, por-talhão, **versões**, **versão oficial** + **profundidade
oficial**) → mapa com a legenda oficial. Aba "Condutividade" na página do talhão.

### Fase C2 — Limpeza + Assistente + Krigagem manual (PENDENTE)
- **Limpeza** antes da interpolação reusando o pipeline da colheita (`backend/colheita.py`):
  Limpeza Global 85% (editável), Local/MapFilter 15% (10% talhões pequenos), Raio 100 m.
  Filosofia: remover só ruído de aquisição, preservar a variabilidade natural.
- **Assistente inteligente**: sugere Global/Local/Raio pelo tamanho do talhão (aceitar/editar).
- **Krigagem Modo 2 (manual)**: modelo/alcance/pepita/patamar/pixel/raio/vizinhos/anisotropia/suavização.
- **Percentual removido** entra no Índice de Qualidade. Export **GeoTIFF**.

### Fase C3 — Vetorização no MEAP + Comparação de Zonas (PENDENTE)
- EC oficial vira **fonte do MEAP** (`lib/meap/gerar.ts`, como o NDVI).
- Fluxo de vetorização: profundidade/interpolação → nº de zonas → vetorizar → área mínima →
  suavizar → remover ilhas → unir polígonos semelhantes → salvar. Métodos de classificação:
  Quantis / Intervalos Iguais / Jenks / Personalizado.
- **Comparação "Zona só Condutividade × Zona Multivariável"** (divisas, similaridade, área/zona,
  nº de polígonos, sobreposição com produtividade/fertilidade/textura/altimetria/NDVI) reusando
  o comparador universal + MEAP 13.00A.

### Fase C4 — Histórico avançado + Laboratório de Zonas (PENDENTE)
Histórico completo de versões (Rasa/Profunda, krigagem auto/manual, zona EC/multivariável),
nunca sobrescrever sem confirmação. Integração Relatórios/Layouts. **Visão futura**:
Laboratório de Zonas de Manejo (testar combinações de variáveis, pesos, comparar algoritmos,
validar cientificamente a melhor configuração por talhão).
