## 2. Atores e papéis

| Ator | Responsabilidade no MEAP |
|---|---|
| **Agrônomo / Consultor** | Dispara geração, revisa, ajusta o modelo operacional, publica a Zona Oficial. |
| **Amostrador / Campo** | Fornece evidências (amostragem, colheita) — entradas, não opera o MEAP. |
| **Administrador da Empresa** | Define pesos-padrão de camadas, paletas de classes e políticas de consolidação. |
| **Sistema (MEAP Engine)** | Calcula modelo matemático, convergência, confiabilidade, maturidade. |

Escopo multi-tenant: todo dado do MEAP é particionado por **empresa** (consultoria) e
**cliente/produtor**, herdando o modelo de `lib/empresa.ts`.

## 3. Entidades de domínio

```
AmbienteProdutivo (1 talhão · 1 modelo vigente)
 ├── VersaoModelo (N — histórico permanente)
 │     ├── parametros (camadas, pesos, nº de classes, seed)
 │     ├── metricas (convergência, confiabilidade, estado do conhecimento)
 │     └── ZonaDeManejo (N por versão)
 │           ├── geometria (operacional) + geometria (matemática)
 │           ├── DNAZona (assinatura estável)
 │           ├── metricas da zona (convergência, confiabilidade, produtividade média)
 │           └── Evidencias[] (referências às camadas que a moldaram)
 ├── CamadaOficial (N — fontes elegíveis)
 └── Prontuario (visão agregada por zona, derivada — não persistida em duplicidade)
```

### 3.1 Definições

- **AmbienteProdutivo** — o "ser vivo" do talhão: o conjunto evolutivo de zonas. Um talhão tem
  **um** ambiente produtivo ativo, que acumula versões.
- **VersaoModelo** — uma fotografia imutável do modelo num instante (o `v5 (12/05/2025)` da tela).
- **ZonaDeManejo** — uma zona dentro de uma versão (Z01..Z05 da tela).
- **DNAZona** — assinatura agronômica que dá **identidade persistente** à zona entre versões.
- **CamadaOficial** — uma fonte de dados validada e elegível (fertilidade, produtividade, …).
- **Evidencia** — o vínculo "esta zona foi moldada por esta camada/dado nesta versão".
- **Prontuario** — ficha consolidada por zona (DNA + histórico + evidências + métricas). Derivada.

## 4. Modelo de dados (Firestore / localStorage)

Padrão de chaves do app: prefixo `inv_`. Coleções de docs grandes seguem o padrão de
`lib/cloud.ts` (espelho opt-in; rasters comprimidos). Migração futura para Postgres/PostGIS
prevista — os tipos abaixo são agnósticos de backend.

### 4.1 Coleções

| Coleção | Conteúdo | Cardinalidade |
|---|---|---|
| `inv_meap_ambientes` | `AmbienteProdutivo` (1 por talhão) | pequena |
| `inv_meap_versoes` | `VersaoModelo` (histórico) | média, cresce sempre |
| `inv_meap_zonas` | `ZonaDeManejo` por versão | média |
| `inv_meap_dna` | `DNAZona` (linhagem persistente da zona) | pequena |
| `inv_meap_rasters` | grids/rasters intermediários (gzip + Storage) | grande (blobs) |

### 4.2 `AmbienteProdutivo`

```ts
interface AmbienteProdutivo {
  id: string;                 // = talhaoId (1:1)
  talhaoId: string;
  empresaId: string;
  clienteId: string;
  versaoVigenteId: string | null;   // aponta para a VersaoModelo oficial
  estado: 'em-formacao' | 'em-consolidacao' | 'consolidada';
  criadoEm: string;           // ISO
  atualizadoEm: string;       // ISO
}
```

### 4.3 `VersaoModelo`

```ts
interface VersaoModelo {
  id: string;
  ambienteId: string;
  numero: number;             // 1,2,3… → rótulo "v5"
  dataReferencia: string;     // "12/05/2025"
  origem: 'manual' | 'novo-dado' | 'reprocesso-agendado';
  parametros: ParametrosModelo;      // ver 13.02 §Parametrização
  metricas: MetricasModelo;
  zonaIds: string[];          // zonas desta versão
  oficial: boolean;           // congelada/publicada?
  criadoEm: string;
}

interface MetricasModelo {
  convergencia: number;             // 0..1 → "76%"
  mudancaEntreVersoes: number;      // 0..1 → "2,8%"
  confiabilidade: 'baixa' | 'media' | 'alta';
  estadoConhecimento: 'insuficiente' | 'inicial' | 'bom' | 'excelente';
  numClasses: number;               // 5 na tela
  reducaoVariancia: number;         // 0..1 — qualidade do zoneamento (13.02 §9.6)
  cvMedioIntraZona: number;         // CV (%) médio intra-zona (variável de validação)
}
```

### 4.4 `ZonaDeManejo`

```ts
interface ZonaDeManejo {
  id: string;
  versaoId: string;
  ambienteId: string;
  dnaId: string;              // linhagem persistente (3.x → DNAZona)
  ordem: number;              // 1..N (Z01..Z05)
  rotulo: string;             // "Zona 01 - Alto Potencial"
  classePotencial: 'alto' | 'medio-alto' | 'medio' | 'medio-baixo' | 'baixo';
  cor: string;                // herdada da paleta da Biblioteca

  geometriaOperacional: GeoJSON.Polygon | GeoJSON.MultiPolygon;  // publicável
  geometriaMatematica:  GeoJSON.Polygon | GeoJSON.MultiPolygon;  // bruta (rastreio)

  areaHa: number;             // 12,45
  percTalhao: number;         // 0,239 → "23,9%"

  metricas: MetricasZona;
  evidencias: Evidencia[];
}

interface MetricasZona {
  convergencia: number;             // 0..1 → "82%"
  confiabilidade: 'baixa' | 'media' | 'alta';
  produtividadeMedia3a: number | null;   // 78,2 (sc/ha)
  potencialProdutivo: 'alto' | 'medio-alto' | 'medio' | 'medio-baixo' | 'baixo';
  atributosResumo: Record<string, number>; // ex.: { argilaMedia: 42 }

  // Homogeneidade interna (13.02 §9.5). CV calculado na ESCALA ORIGINAL dos
  // atributos (nunca no índice normalizado P(x)). Gerado junto com a versão.
  cvValidacao: number;              // CV (%) da variável de validação — headline da zona
  variavelValidacao: string;        // 'produtividade' (≥3 safras) | atributo de maior peso
  cvPorAtributo: Record<string, number>;       // CV (%) por atributo → prontuário
  homogeneidade: 'alta' | 'media' | 'baixa';   // faixa legível derivada do CV
}

interface Evidencia {
  camadaId: string;
  tipo: 'fertilidade' | 'amostragem' | 'produtividade' | 'ndvi' | 'compactacao' | 'topografia';
  peso: number;               // contribuição no modelo (0..1)
  dataDado: string;           // ISO da coleta/importação
}
```

### 4.5 `CamadaOficial`

```ts
interface CamadaOficial {
  id: string;
  ambienteId: string;
  tipo: Evidencia['tipo'];
  fonte: string;              // id da importação de origem (lab, grade, mapa de colheita…)
  rasterId: string | null;    // referência em inv_meap_rasters (se contínua)
  qualidade: 'baixa' | 'media' | 'alta';   // ver critérios 13.02
  elegivel: boolean;          // passou no gate de qualidade?
  dataDado: string;
}
```

## 5. DNA da Zona

O **DNA** é o que permite dizer "a Zona 01 da v5 é a evolução da Zona 01 da v4", mesmo com a
geometria ligeiramente diferente. Sem DNA, cada versão seria um conjunto de zonas estranhas entre si.

```ts
interface DNAZona {
  id: string;
  ambienteId: string;
  linhagem: number;           // identidade estável (a "Zona 01" ao longo do tempo)

  // Assinatura agronômica (vetor de características estáveis, normalizado 0..1):
  assinatura: {
    potencialProdutivo: number;
    fertilidadeDominante: number;
    argila: number;
    posicaoPaisagem: number;  // topo/meia-encosta/baixada
    estabilidadeTemporal: number;
  };

  centroideHistorico: [number, number];   // âncora espacial média
  primeiraVersao: number;     // quando surgiu
  versoesPresente: number[];  // em quais versões existiu
  maturidade: 'em-formacao' | 'em-consolidacao' | 'consolidada';
}
```

**Matching entre versões** (regra): ao gerar a versão N, cada zona nova é casada com um DNA
existente por **maior similaridade** (distância da assinatura + proximidade do centróide,
acima de um limiar). Sem casamento ⇒ nasce um DNA novo (zona realmente nova). DNA sem zona na
versão atual ⇒ marcado como "dormente" (não excluído — memória). Algoritmo detalhado: 13.02.

## 6. Identidade e versionamento

Regras invioláveis (decorrem do pilar *memória permanente*):

1. **VersaoModelo é imutável.** Publicada ou não, nunca é editada — gera-se uma nova versão.
2. **Toda versão referencia a anterior** (`numero - 1`) para cálculo de convergência/mudança.
3. **Apenas uma versão é `oficial`** por ambiente em dado momento (`versaoVigenteId`).
4. **Rollback** = republicar uma versão antiga como oficial (não apaga as posteriores).
5. **DNA atravessa versões**; geometria e métricas pertencem à versão.

---

## 7. Armazenamento de rasters

- Grids contínuos (saída do interpolador, mapas intermediários) são **Float32** comprimidos
  (gzip), no padrão já usado pela Fertilidade (`lib/raster.ts`, `cloudSalvarMapa`).
- Vão para `inv_meap_rasters` / Storage, **nunca** inline no doc da versão (evita estourar
  limites de documento e tráfego).
- A `ZonaDeManejo` guarda apenas **vetores** (GeoJSON) — leves e versionáveis.

## 8. Máquina de estados (ambiente e zona)

```
            ┌─────────────┐   convergência↑ e dados↑   ┌──────────────────┐
  início →  │ Em Formação │ ─────────────────────────→ │ Em Consolidação  │
            └─────────────┘                            └──────────────────┘
                                                              │ convergência alta e estável
                                                              ▼
                                                       ┌──────────────┐
                                                       │ Consolidada  │
                                                       └──────────────┘
```

Critérios numéricos de transição: definidos na **13.03 §Maturidade**. Esta parte só fixa os
estados e que a transição é **automática e auditável** (registrada na versão que a disparou).

---

## 9. Regras de integridade

- Soma das áreas das zonas de uma versão = área do talhão (tolerância ≤ 0,5%).
- `percTalhao` sempre derivado de `areaHa / areaTalhao` (nunca digitado).
- Toda zona tem `dnaId` válido; todo `rasterId` referenciado existe.
- `versaoVigenteId` aponta para versão com `oficial = true`.
- Evidência sempre referencia uma `CamadaOficial` com `elegivel = true`.
