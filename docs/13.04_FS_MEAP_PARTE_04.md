## 3. Especificação por componente

### 3.1 `MeapZonasCard`
- Lista ordenada Z01..ZN: cor, rótulo (`"Zona 01 - Alto Potencial"`), área (ha), % do talhão.
- Badge de estado do ambiente: **Em Formação / Em Consolidação / Consolidada**.
- Clique numa zona → seleciona no mapa e no `MeapZonaInfo`.

### 3.2 `MeapConvergenciaCard`
- "Convergência do Modelo — NN%" (barra de progresso).
- "Mudança entre últimas versões: N,N%".
- "Versão atual: vN (dd/mm/aaaa)".
- Link **"Ver histórico de versões"** → `MeapHistorico`.

### 3.3 `MeapStatusCard`
- **Índice de Confiabilidade:** Baixa/Média/Alta (cor).
- **Estado do Conhecimento:** Insuficiente/Inicial/Bom/Excelente.

### 3.4 `MeapZonaInfo` (painel direito)
- Zona selecionada: rótulo + classe.
- Área (ha + %), Produtividade Média (3 anos), Potencial Produtivo, Argila Média (e demais
  `atributosResumo`), Índice de Convergência (zona), Índice de Confiabilidade,
  **Homogeneidade interna (CV)** — selo Alta/Média/Baixa + valor.
- Botão **"Ver prontuário da zona"** → `MeapProntuario`.

### 3.5 `MeapProntuario`
- Render dos 6 blocos do prontuário (13.03 §7), incluindo a linha do tempo da zona e a
  explicabilidade legível.

### 3.6 `MeapHistorico`
- Tabela/linha do tempo de `VersaoModelo`: número, data, origem, convergência, oficial?
- Ações: **Comparar** duas versões; **Tornar oficial** (rollback — papel agrônomo).

### 3.7 Mapa — controles `Comparar` e `Histórico`
- **Comparar:** modo lado-a-lado / swipe entre duas versões de zonas.
- **Histórico:** *scrubber* temporal por versão sobre o mesmo talhão.

## 4. Fluxos de usuário

### 4.1 Gerar zonas (primeira vez)
```
Resumo → "Gerar zonas (MEAP)" → escolhe parâmetros (ou usa default da empresa)
     → Engine roda pipeline (13.02) → v1 criada (Em Formação)
     → agrônomo revisa → publica como Zona Oficial
```

### 4.2 Refinar com novo dado
```
Nova evidência elegível chega (lab/colheita/NDVI) → sistema sinaliza "há dado novo"
     → agrônomo dispara refinamento → nova versão (13.02+13.03)
     → compara com a anterior → publica se aprovar
```

### 4.3 Consolidar
```
Convergência alta e estável → maturidade promove p/ Consolidada (automático, auditável)
```

### 4.4 Comparar / rollback
```
Histórico → seleciona v(n) e v(m) → Comparar → (opcional) Tornar v(k) oficial
```

## 5. Integração com os módulos da plataforma

| Módulo | Direção | Interface real |
|---|---|---|
| **Fertilidade** | entrada (camada) | `lib/fertilidade.ts`, `FertilidadeSection.tsx`, grids em Storage |
| **Amostragem** | entrada (camada) | `lib/grid.ts`, `AmostragemModulo.tsx` |
| **Produtividade** | entrada (camada, média 3 anos) | mapas de colheita *(em desenvolvimento)* |
| **NDVI / Satélite** | entrada (camada) | aba NDVI/Satélite |
| **Compactação** | entrada (camada) | `lib/compactacao.ts`, `CompactacaoSection.tsx` |
| **Biblioteca / Legendas** | configuração | `lib/biblioteca.ts`, `lib/legendas.ts` (`PARES_OFICIAIS_5`, paletas) |
| **Recomendações / Cenários** | **saída** (consome zona oficial) | `lib/recomendacao/`, `RecomendacaoSection.tsx` |
| **Relatórios** | **saída** | `lib/relatorioDados.ts`, `lib/relatorioFertilidade.ts`, `GeradorRelatorios.tsx` |
| **Empresa / multi-tenant** | escopo | `lib/empresa.ts` |

Regra de ouro: o MEAP **lê** camadas oficiais e **escreve** a Zona Oficial. Nunca edita dados de
outro módulo.

## 6. Contratos de serviço

### 6.1 Front-end (TypeScript) — `lib/meap/`
Proposta de libs (a criar; **não implementar agora**):

```ts
// lib/meap/engine.ts
gerarVersao(ambienteId: string, params?: Partial<ParametrosModelo>): Promise<VersaoModelo>;
refinarComNovaEvidencia(ambienteId: string): Promise<VersaoModelo>;
publicarOficial(ambienteId: string, versaoId: string): Promise<void>;   // ou rollback

// lib/meap/metricas.ts
calcularConvergencia(atual: VersaoModelo, anterior: VersaoModelo | null): MetricasModelo;

// lib/meap/dna.ts
casarDNA(zonas: ZonaDeManejo[], dnas: DNAZona[], params): MatchResult;

// lib/meap/store.ts  → segue padrão store.ts/cloud.ts (inv_meap_*)
```

### 6.2 Backend (Python `backend/`) — cálculo pesado
Endpoints (estendem o serviço de interpolação atual):

```
POST /meap/interpolar      → krigagem/IDW + validação cruzada por camada
POST /meap/zonear          → fusão + clusterização + vetorização → zonas (geometria matemática)
```

Entradas/saídas em GeoJSON + grids Float32 gzip (padrão `lib/fertilidade.ts`). O front orquestra,
o backend faz o numérico (NumPy/PyKrige/SciPy/Shapely).

## 7. Estados de UI e feedback

- **Sem zonas ainda:** call-to-action "Gerar zonas (MEAP)" (nunca gerar automaticamente sem
  comando — coerente com a correção do grid de amostragem feita em v0.64.1).
- **Há dado novo:** selo "Atualização disponível" no card de convergência.
- **Processando:** estado de carregamento explícito (o backend pode demorar).
- **Confiabilidade baixa / Estado insuficiente:** aviso honesto ("poucos dados — zona provisória").
